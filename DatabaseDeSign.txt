# DATABASE DESIGN — BLOCKCHAIN-BASED SECURE E-VOTING WITH PAILLIER HOMOMORPHIC ENCRYPTION

# 1. DESIGN GOALS

Hệ thống được thiết kế để đáp ứng:

* secure e-voting
* anonymity
* one-vote-per-user
* homomorphic tallying
* blockchain auditability
* threshold trust
* public/private election
* research-grade extensibility

Kiến trúc chia thành:

* Off-chain layer (MongoDB)
* Cryptographic layer
* Blockchain audit layer

---

# 2. DATABASE ARCHITECTURE

MongoDB chỉ lưu:

* application data
* encrypted ballots
* user/account data
* election metadata
* audit logs

Blockchain chỉ lưu:

* election hash
* ballot hash
* result hash
* immutable audit records

---

# 3. COLLECTION OVERVIEW

| Collection          | Purpose                       |
| ------------------- | ----------------------------- |
| users               | Tài khoản hệ thống            |
| elections           | Cuộc bầu cử                   |
| candidates          | Ứng viên                      |

| refresh_tokens      | JWT refresh session           |
| notifications       | Email/system notification     |
| election_results    | Final tally                   |
| zk_proofs(optional) | ZKP verification              |

# 18. SECURITY DESIGN SUMMARY

| Feature           | Mechanism            |
| ----------------- | -------------------- |
| Password security | bcrypt               |
| Vote privacy      | Paillier HE          |
| One-vote-per-user | token invalidation   |
| Auditability      | blockchain hashes    |
| Integrity         | SHA256 + signatures  |
| Replay protection | nonce                |
| Anonymous voting  | voting token         |
| Distributed trust | threshold decryption |
| Tamper resistance | immutable chain      |
| Vote validity     | ZKP                  |
